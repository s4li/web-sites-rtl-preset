# RTL Preset — Chrome Extension

## صورت مسئله

خیلی از سایت‌های غیرفارسی (مثل Claude.ai، ChatGPT، GitHub و ...) لایوتشون چپ‌به‌راست (LTR) هست. وقتی کاربر فارسی‌زبان با این سایت‌ها کار می‌کنه، خوندن و تایپ متن فارسی توی یه محیط LTR ناراحت‌کننده‌ست — متن‌ها از سمت اشتباه شروع می‌شن، چشم مجبوره مسیر غیرطبیعی طی کنه و تجربه کاربری افت می‌کنه.

**مشکلات مشخص:**

1. **نبود کنترل سراسری:** مرورگر امکان تعریف لیست سایت‌هایی که باید RTL بشن رو نداره. کاربر مجبوره هر بار دستی direction رو عوض کنه یا از DevTools استفاده کنه.

2. **خراب شدن UI بعد از RTL:** وقتی یه سایت LTR رو به‌زور RTL می‌کنی، لایوت به هم می‌ریزه — منوها روی هم میفتن، فاصله‌ها (margin/padding) اشتباه می‌شن، المنت‌های absolute/fixed جاشون عوض نمی‌شه، و border-radius و float‌ها آینه نمی‌شن.

3. **نبود کنترل جزئی:** گاهی نمی‌خوای کل صفحه RTL بشه، فقط یه بلاک خاص (مثلاً یه باکس چت یا یه پاراگراف) نیاز داره direction عوض بشه. ابزار ساده‌ای برای این کار وجود نداره.

4. **فونت نامناسب:** خیلی از سایت‌ها فونت مناسب فارسی ندارن. متن فارسی با فونت پیش‌فرض سایت نمایش داده می‌شه که معمولاً قشنگ نیست.

## راه‌حل

یه Chrome Extension ساختیم به اسم **RTL Preset** که همه این مشکلات رو توی یه ابزار حل می‌کنه:

### ۱. لیست سایت‌های RTL (Preset)
کاربر از طریق popup اکستنشن، سایت‌هایی که می‌خواد همیشه RTL باشن رو تعریف می‌کنه (مثلاً `claude.ai`). از اون به بعد هر بار که وارد اون سایت بشه، لایوت اتوماتیک RTL می‌شه. لیست با `chrome.storage.sync` ذخیره می‌شه و بین دستگاه‌ها هم sync می‌شه.

### ۲. موتور اصلاح UI خودکار
یه toggle به اسم «اصلاح UI» داریم که وقتی فعال بشه، تمام المنت‌های صفحه رو اسکن می‌کنه و مشکلات رایج RTL رو اتوماتیک درست می‌کنه:

| چی رو درست می‌کنه | چطوری |
|---|---|
| Position (absolute/fixed/sticky) | مقدار `left` و `right` رو جابجا می‌کنه |
| Margin | margin-left و margin-right رو عوض می‌کنه |
| Padding | padding-left و padding-right رو عوض می‌کنه |
| Float | `float: left` ↔ `float: right` |
| Border | border-left و border-right رو swap می‌کنه |
| Border-radius | گوشه‌های بالا-چپ/راست و پایین-چپ/راست رو آینه می‌کنه |
| Transform | مقدار `translateX` رو معکوس می‌کنه |
| Background-position | `left` ↔ `right` |

- پردازش **chunk-based** هست (با `requestAnimationFrame`) تا صفحه فریز نکنه.
- از **MutationObserver** استفاده می‌کنه تا المنت‌هایی که بعداً به DOM اضافه می‌شن (مثل محتوای lazy-load یا منوهای dropdown) هم اصلاح بشن.
- وقتی خاموش بشه، تمام استایل‌های اصلی restore می‌شن (از `WeakMap` برای ذخیره مقادیر اولیه استفاده شده).

### ۳. تغییر جهت المنت‌های تکی (Alt+Click)
بدون باز کردن popup، کاربر می‌تونه:
- **Alt نگه داره** → المنت زیر موس هایلایت آبی می‌شه
- **Alt + Arrow Up/Down** → بین المنت والد و فرزند حرکت کنه
- **Alt + Click** → جهت اون المنت خاص رو toggle کنه (RTL ↔ LTR)

تغییرات المنت‌های تکی هم **ذخیره می‌شن** (با CSS selector) و بعد از رفرش صفحه بازیابی می‌شن.

### ۴. فونت فارسی
برای هر سایت می‌شه فونت فارسی انتخاب کرد:
- **Vazirmatn** (وزیرمتن) — از Google Fonts
- **Noto Sans Arabic** — از Google Fonts
- **Sahel** (ساحل) — از jsDelivr CDN
- **Samim** (صمیم) — از jsDelivr CDN

### ۵. شورتکات کیبورد
`Ctrl+Shift+R` برای toggle سریع RTL کل صفحه فعلی، بدون نیاز به باز کردن popup.

### ۶. Badge روی آیکون
وقتی سایت فعلی توی لیست RTL باشه، روی آیکون اکستنشن یه badge قرمز با نوشته `RTL` نشون داده می‌شه.

### ۷. Import/Export
تمام تنظیمات (لیست سایت‌ها، فونت‌ها، تغییرات المنت‌ها، وضعیت UI Fix) قابل خروجی‌گیری به JSON و وارد کردن هستن — برای بکاپ یا انتقال به سیستم دیگه.

## معماری فنی

```
manifest.json          → تنظیمات اکستنشن (Manifest V3)
background.js          → Service Worker — badge + شورتکات
content.js             → Content Script — اعمال RTL + اصلاح UI + Alt+Click
popup.html/css/js      → رابط کاربری popup
icons/                 → آیکون‌های اکستنشن
```

**Storage keys:**
- `rtlSites` → آرایه hostname‌ها (`["claude.ai", "github.com"]`)
- `rtlFonts` → آبجکت فونت به‌ازای هر سایت (`{"claude.ai": "vazirmatn"}`)
- `rtlUIFix` → آبجکت وضعیت اصلاح UI (`{"claude.ai": true}`)
- `rtlElements` → آبجکت تغییرات المنت‌های تکی (`{"claude.ai": {"#msg": "rtl"}}`)

## محدودیت‌ها و نکات

- **اصلاح UI صد در صد نیست:** بعضی سایت‌ها ساختار CSS پیچیده‌ای دارن که اصلاح کامل خودکار ممکن نیست. ولی برای اکثر سایت‌ها نتیجه قابل قبوله و بقیه رو می‌شه با Alt+Click دستی درست کرد.
- **Performance:** اسکن DOM روی صفحات خیلی سنگین (هزاران المنت) ممکنه یه لحظه delay ایجاد کنه. برای همین از chunk processing استفاده شده.
- **ذخیره Alt+Click:** از CSS selector برای شناسایی المنت‌ها استفاده می‌شه. اگه ساختار DOM سایت تغییر کنه، ممکنه بعضی ذخیره‌ها دیگه match نکنن.
